StudentOS — Auth Debug

This page helps determine whether the Supabase auth redirect created a session and whether the SIGNED_IN state can be observed.

Instructions

  1. After completing Google sign-in and being redirected back, open this page (bookmark: /auth-debug.html on your site) in the SAME tab.
  2. The page will read the URL hash tokens, call Supabase /auth/v1/user using the access_token, and poll localStorage for the Supabase session token.
  3. Watch the console for logs: EVENT SIGNED_IN (via localStorage detection), SESSION (parsed hash), USER (fetched from Supabase).
